# Assets on the Network

Refactor 2026 introduces a completely new digital asset system that makes tokens and NFTs first-class citizens of the Saito network. Instead of existing as smart contracts that every application must integrate independently, digital assets are understood by the blockchain itself. Wallets, marketplaces, games, publishing platforms, and every other Saito application automatically recognize and support the same assets without additional development.

Whether you're creating a new cryptocurrency, issuing collectible NFTs, selling digital products, managing subscriptions, or building an online game, the same asset system powers every application on the network.

## Native Digital Assets

Creating a token on Saito is now as simple as creating a wallet or sending a payment. Every token is issued through the consensus layer and immediately becomes compatible with the rest of the ecosystem.

Applications don't need to write custom integrations to support new assets. Once an asset exists, it can be transferred, traded, stored, displayed, and used anywhere on the network.

This dramatically lowers the cost of building applications while ensuring that users have a consistent experience regardless of where their assets are used.

## More Than Traditional NFTs

Saito's NFTs are not limited to representing static collectibles.

Because they exist as native blockchain objects, NFTs can represent almost any form of digital ownership, including:

- Collectibles
- Game items
- Tickets
- Memberships
- Access keys
- Software licenses
- Domain names
- Identity credentials
- Digital merchandise
- Subscription rights

Applications determine what an NFT represents, but every application understands how to store, transfer, and verify ownership.

## Split and Merge Assets

One of the biggest additions in Refactor 2026 is support for splitting and merging NFTs.

Digital assets no longer need to remain fixed once they are created.

An NFT can be divided into smaller ownership units or combined with other compatible assets, allowing developers to create far richer ownership models than are possible with traditional non-fungible tokens.

This enables applications such as:

- Fractional ownership
- Stackable game resources
- Bundled digital products
- Multi-license software
- Shared ownership rights

These operations are supported by the platform itself rather than implemented separately by every application.

## One Wallet for Everything

The redesigned wallet understands every native asset automatically.

Users can hold cryptocurrencies, NFTs, subscriptions, game assets, access keys, and other digital property in the same interface without installing additional plugins or switching between applications.

Applications don't need to manage their own asset storage. They simply ask the wallet for the assets they need.

## Assets That Move Between Applications

One of the advantages of building assets into the network is that ownership becomes portable.

An NFT created inside one application can immediately be recognized by another.

A game can issue items that appear in the wallet.

A publishing platform can create access passes that can be traded through the marketplace.

A developer can issue software licenses that every Saito application already knows how to transfer.

Instead of every application creating its own isolated asset ecosystem, Refactor 2026 provides a common foundation that every application shares.

## Built for Programmable Ownership

Native assets become even more powerful when combined with RustScript.

Tokens and NFTs can be locked into programmable scripts that define how ownership changes over time.

Assets can require multiple signatures before transferring.

They can enforce royalty payments automatically.

They can remain locked until specific conditions are satisfied.

They can represent subscriptions, escrow agreements, collaborative ownership, or application-specific rules.

Because these capabilities are provided by the blockchain itself, developers do not need to recreate them inside every application.

## Ready for Commerce

Every native asset is automatically compatible with the Saito Store and the network's asynchronous marketplace.

Users can buy and sell digital goods without needing to coordinate with another party in real time.

Applications don't need to build their own marketplace infrastructure or payment systems. Assets issued anywhere on the network can immediately participate in decentralized commerce.

## Digital Ownership as Infrastructure

Traditional blockchain ecosystems often treat tokens, NFTs, marketplaces, wallets, and applications as separate systems that must all be connected manually.

Refactor 2026 takes a different approach.

Digital ownership is part of the platform itself.

Applications inherit the ability to issue, manage, transfer, protect, and trade digital assets simply by building on Saito.

Instead of every project reinventing the same infrastructure, developers can focus on building new experiences while the network handles ownership, portability, and commerce automatically.
